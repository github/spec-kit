Top-level documents are not sufficient for full onboarding. Before editing, read the relevant subsystem documentation for the surface you plan to modify.

Use the following routing guide.

| If you plan to change...                                 | Read...                                                                                        |
| -------------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| Core workflow templates and generated artifact structure | The relevant documents under `templates/` and any linked workflow-specific documentation       |
| Validation behavior or test expectations                 | [TESTING.md](./TESTING.md) and any related subsystem-specific testing documentation            |
| Extensions                                               | The documentation under `extensions/`, including extension publishing and maintenance guidance |
| Presets                                                  | The documentation under `presets/`, including preset publishing and maintenance guidance       |
| User-facing workflow semantics                           | The relevant templates, [README.md](./README.md), and any conceptual workflow documentation    |
| Installation, packaging, or bootstrap behavior           | The relevant CLI/package code, scripts, and associated development or publishing documentation |
