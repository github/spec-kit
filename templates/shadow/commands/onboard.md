# Project Onboarding

Generate onboarding materials for new team members or AI agents.

## Usage

```bash
{SHADOW_PATH}/scripts/bash/onboard{SCRIPT_EXT}
```

## What This Creates

### Project Overview
- Purpose and goals
- Key features
- Technology stack
- Architecture overview

### Getting Started
- Prerequisites
- Setup instructions
- Build and run
- Development workflow

### Code Organization
- Directory structure
- Module descriptions
- Important files
- Naming conventions

### Development Practices
- Coding standards
- Testing approach
- Code review process
- CI/CD pipeline

### Resources
- Documentation links
- Communication channels
- Key contacts
- Learning materials

## Output

Generates:
```
docs/ONBOARDING.md
.ai/quick-start.md
```

## Use Cases

### New Developers
Get up to speed quickly

### AI Agents
Provide necessary context

### Contractors
Understand project quickly

### Open Source Contributors
Lower barrier to contribution

## What to Include

1. **Quick wins** - Small tasks to start with
2. **Code walkthrough** - Important files to review
3. **Common tasks** - How to add features, fix bugs
4. **Gotchas** - Common pitfalls and how to avoid them
5. **Resources** - Where to find help

## Best Practices

- Keep onboarding docs updated
- Include visual aids
- Test with new team members
- Iterate based on feedback
- Maintain FAQ section
