# Generate Quality Checklist

Create a quality checklist to validate requirements completeness, clarity, and consistency.

## When to Use

Run this command **after** `/plan` to ensure quality before implementation.

## What This Generates

### Specification Quality Checks
- [ ] All requirements clearly defined
- [ ] Success metrics identified
- [ ] Acceptance criteria complete
- [ ] Dependencies documented
- [ ] Risks with mitigation

### Plan Quality Checks
- [ ] All requirements addressed
- [ ] Tasks appropriately sized
- [ ] Dependencies identified
- [ ] Timeline is realistic
- [ ] Resources allocated

### Implementation Readiness
- [ ] Prerequisites met
- [ ] Tools available
- [ ] Team has skills
- [ ] Testing strategy defined

## Output

Checklist saved to:
```
.checklists/<feature-name>-checklist.md
```

## Usage

1. Generate checklist with this command
2. Review each item systematically
3. Check off completed items
4. Address any gaps or issues
5. Get sign-off from stakeholders

## Checklist Categories

- Specification completeness
- Plan coverage
- Task actionability
- Implementation readiness
- Quality assurance
- Documentation
- Sign-off requirements

## Benefits

- Catch issues early
- Ensure nothing is forgotten
- Provide confidence to proceed
- Create paper trail
- Facilitate reviews
